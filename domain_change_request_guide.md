# 舊網址轉接修改指南（Email 範本與操作流程）

當我們的網站網址是向先前合作的廠商（如網頁代理商或舊資訊廠商）購買，且公司手邊沒有 DNS 控制台密碼時，請參考以下兩種解決方案進行網址轉接。

---

## 💡 解決方案 A：委託前合作廠商修改（最推薦、免帳密）

這是最安全且快速的方法。請直接複製下方的 Email 範本，填入您的「網域」與「公司名稱」後，寄給前一家公司的客服或工程人員處理。

### ✉️ 寄給前廠商的修改申請 Email 範本

```text
主旨：【網站變更協助】請協助更新我司網域（ 您的網址，例如 yourdomain.org ）的 DNS 指向設定

您好：

我司目前正在進行網站伺服器主機的移轉作業，需請貴司工程團隊協助調整我們網址的 DNS 指向解析。

請幫我們新增或修改以下兩條 DNS 記錄：

1. 根網域 A 記錄（A Record）：
   - 主機記錄 (Name / Host)：@（或留空）
   - 指向內容 (Value / IP)：76.76.21.21

2. 子網域 CNAME 記錄 (CNAME Record)：
   - 主機記錄 (Name / Host)：www
   - 指向內容 (Value / Target)：cname.vercel-dns.com

※ 重要聲明：請務必保留我司現有的電子郵件（MX 記錄、SPF 記錄等）原有設定，切勿做任何變更或刪除，以免影響公司 Email 的日常收發。

設定完成後，煩請回信告知我們以便進行連線測試，非常感謝您的協助！

您的公司名稱 敬上
```

---

## 💡 解決方案 B：要求轉移/拿回網址管理平台權限

如果您主管希望未來能自主控制網址，不需透過外部廠商，可向前廠商提出移交要求：

1. **詢問註冊商平台**：確認當初網址是在哪一個網址註冊商（例如 GoDaddy、中華電信/HiNet 網域註冊、PChome 買網址等）購買的。
2. **要求轉移所有權/帳密**：要求前廠商將該平台的**管理帳號密碼轉交給您**，或者將該網址轉移至您自建的網址商帳號中。
3. **自行設定**：成功登入網址商控制台後，找到「DNS 記錄管理」，手動加入上述範本中的 **A 記錄** 與 **CNAME 記錄** 即可。

## 🔑 技術常識備忘：指向的 IP 位址來源

我們不需要另外去申請或尋找專屬 IP：
* **`76.76.21.21`** (A 記錄)
* **`cname.vercel-dns.com`** (CNAME 記錄)

這兩組是指向 Vercel 雲端伺服器的**全球固定專用地址**。任何網址只要指向這兩個位置，就能自動讀取在 Vercel 部署好的新網頁，因此直接提供這組資訊給工程人員即可。
